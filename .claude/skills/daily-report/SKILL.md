## 프로젝트 해석

1. `--project <key>` 인자가 있으면 그 프로젝트 사용
2. 없으면 `loregist projects --current` (cwd 기준 자동 추론)
3. 추론 실패 시 사용자에게 프로젝트 키 질문

`docs_root`: 추론된 프로젝트의 docs_root 값 (`loregist projects --json`으로 확인)

→ loregist/CLAUDE.md "스킬 공통 — 프로젝트 추론 규칙" 적용
작업문서 경로: `{docs_root}/{날짜}/{날짜}.01.작업문서.md`

# 역할

매일 아침/저녁 슬랙 데일리 보고를 자동 생성한다.
**작업문서를 1순위 데이터 소스**로 사용하고, Jira/git은 보조로 참조한다.

# 입력 분석

`$ARGUMENTS`를 파싱한다:

1. **모드**: `morning`(m/아침) | `evening`(e/저녁) | 인자 없음(13시 기준 자동)
2. **기준일**: `YYYY-MM-DD` 형식이 인자에 있으면 해당 날짜를 **기준일**로 사용. 스킬 내 모든 날짜 참조(어제/오늘, 파일 저장 경로, git log 범위, symlink)를 기준일 기준으로 재설정한다. 인자 없으면 실제 오늘.
3. **옵션**: `--team`(팀 리포 커밋 포함), `--copy`(클립보드 복사), `--init`(작업문서 자동 생성), `--no-ticket`(티켓 번호 생략), `--dual`(슬랙용+상세 2단계 생성)

# 데이터 수집

## 1단계 (1순위): 작업문서 읽기

경로: `{docs_root}/{날짜}/{날짜}.01.작업문서.md`

**모드 공통**: 어제+오늘 작업문서를 읽는다.
- **"어제 한 일"**: 어제 작업문서의 `오늘 할 일` 항목을 기본 목록으로 사용. 진행률은 `작업 목록` 체크박스에서 산출.
- **"오늘 할 일"**: 오늘 작업문서의 `오늘 할 일` 섹션을 그대로 사용. 진행률은 `작업 목록`에서 산출.

### 필수 섹션 파싱

1. **`## 오늘 할 일`** — 여기에 명시된 항목**만** 보고에 포함. Jira에서 임의로 끌어오지 않음.
2. **`## 작업 목록`** — `[x]`/`[ ]` 파싱하여 진행률 산출. `> 상세:` 링크 주제별 문서의 체크박스도 참조 (인덱스 우선).
3. **`## Jira 상태 불일치 메모`** — 해당 티켓은 Jira 상태 무시, 작업문서 실제 상태를 따름.
4. **`## 외부 블로커`** — 해소된 블로커 있으면 `💡 블로커 해소됨` 경고 출력 (참고 정보).

### 필터링 규칙

CLAUDE.md의 **작업 항목 태그 규칙**에서 "제외"인 항목은 보고에서 제외한다. 추가로:
- `- [-]` 체크박스 (블로킹) → 제외
- 개인 repo 문서 작업 (인프라 접속정보 갱신, 작업문서 정리 등) → 제외. VM 작업, 팀 repo 커밋, 고객 대응만 보고 대상.

### 완료 판정

| 상태 | 진행률 |
|---|---|
| `[완료]` 태그 또는 모든 `[x]` | 100% |
| 일부 체크됨 | `[x]개수 / 전체` (10% 단위 반올림) |
| 체크박스 없음 | Jira 상태 + 커밋으로 추정 (2단계). 근거 없으면 `[??%]` |

## 2단계 (보조): Jira 티켓 조회

JQL: `assignee = currentUser() AND status in ("In Progress", "To Do", "In Review") ORDER BY status ASC, priority DESC`

**용도 한정**: 티켓 번호 매칭, 체크박스 없는 항목 진행률 보조, 작업문서에 없는 티켓 경고 출력(보고에 미포함).
Jira에서 임의로 항목을 추가하지 않는다.

## 3단계 (보조): Git 커밋 이력 수집

```bash
git -C {repo_path} log --oneline --after="어제 00:00" --before="오늘 00:00" --author="$(git config user.email)"
```

**용도 한정**: 작업 내용 요약 보강, 작업문서에 없는 커밋 작업 별도 표시.

# 출력 포맷

```
* 어제 한 일
    - [100%] {TICKET-XX} {완료된 작업 요약}
    - [80%] {TICKET-XX} {진행 중 작업 요약}
* 오늘 할 일
    - [80%] {TICKET-XX} {이어서 할 작업}
    - [0%] {TICKET-XX} {새로 시작할 작업}
```

- `--no-ticket`: 티켓 번호 생략
- 아침/저녁 동일 포맷
- 경고(`⚠ 작업문서에 없는 Jira 티켓`, `⚠ Jira 상태 불일치`, `💡 블로커 해소됨`)는 보고 아래에 별도 참고 출력

# 출력 규칙

1. 필터링 후 남은 항목 그대로 출력. 인위적 병합 금지
2. 항목명 정리: `— {상세}` 접미사, `(MM-DD 이월)` 태그 제거. 나머지 있는 그대로
3. 제외된 항목은 `## 미보고` 섹션에 코드블록으로 보존
4. `--copy`: 결과를 `pbcopy`로 클립보드에 복사

## 화면 출력 포맷

- **`--dual` 모드**: 슬랙용 보고를 **코드블록(` ``` `)으로 감싸서** 화면에 출력한다. 터미널에서 바로 복사할 수 있도록.
- **`--dual` 외**: 코드블럭 없이 플레인 텍스트로 출력 (슬랙 복붙 전용)

# --dual 옵션

Jira 조회 없이 즉시 슬랙용을 생성한 뒤, 이어서 Jira 매칭하여 상세 버전을 추가한다.

**1단계 (즉시)**: 작업문서 + git log만으로 티켓 번호 없는 슬랙용 생성 → `{docs_root}/{오늘날짜}/daily-report.md` 저장 → symlink `{docs_root}/daily-report-latest.md` 갱신 → **코드블록으로 감싸서** 화면 출력 (--copy 시 클립보드 복사)

**2단계 (이어서)**: Jira 조회 → 티켓 번호 매칭 → 기존 파일에 `## 상세 (티켓 번호 포함)` 섹션 추가. 실패 시 슬랙용만 유지.

symlink: `ln -sfn {오늘날짜}/daily-report.md {docs_root}/daily-report-latest.md`

# --init 옵션

오늘 작업문서가 없을 때, 전날 작업문서에서 미완료(`[ ]`) 항목·외부 블로커·Jira 상태 불일치 메모를 이월하여 자동 생성. 이후 보고를 이어서 수행.

# 리포 경로

개인 리포 경로는 현재 프로젝트의 `docs_root` 상위 디렉터리 (`loregist projects --json`으로 확인).
`--team` 옵션 사용 시 팀 리포 경로는 사용자에게 확인 또는 대화 맥락에서 추론한다.

# 제약 조건

1. 코드 변경 없음 — 읽기 전용 (`--init` 작업문서 생성, `--dual` 파일 저장만 예외)
2. 작업문서 없으면 Jira + git log 기반 생성 (경고 포함)
3. Jira 조회 실패 시 작업문서 + git log만으로 생성
4. 과장 표현 금지. **근거 없는 진행률 금지** — 추정 근거 없으면 `[??%]`
