## 프로젝트 해석

1. `--project <key>` 인자가 있으면 그 프로젝트 사용
2. 없으면 `stashdex project current` (cwd 기준 자동 추론)
3. 추론 실패 시 사용자에게 프로젝트 키 질문

`docs_root`: 추론된 프로젝트의 docs_root 값 (`stashdex project list`로 확인)

→ loregist/CLAUDE.md "스킬 공통 — 프로젝트 추론 규칙" 적용

이 스킬의 모든 상대 경로는 `{docs_root}/` 기준이다.

## 크로스 프로젝트 이월 금지

- 7일 역탐색·순번 재채번·외부 블로커 테이블은 해당 project의 `docs_root` 안에서만 동작
- 다른 프로젝트의 `docs_root`를 참조하는 크로스 프로젝트 이월 금지

# 역할

전일(또는 지정일) 작업문서에서 미완료 항목을 추출하여 오늘 작업문서로 이월한다.
이월된 항목에는 원본 날짜 태그를 붙이고, 전일 원본에도 이월 표기를 남긴다.

# 입력 분석

`$ARGUMENTS`를 파싱한다:

1. **소스 날짜** — 첫 번째 인자 (선택)
   - `YYYY-MM-DD` 형식: 해당 날짜 작업문서에서 이월
   - 인자 없음: 오늘 기준 직전 작업문서를 자동 탐색 (전일 없으면 그 전날까지 역순 탐색, 최대 7일)

2. **옵션**:
   - `--dry-run`: 실제 파일 수정 없이 이월 대상 항목만 출력

# 경로 규칙

| 구분 | 경로 |
|---|---|
| 인덱스(작업문서) | `{docs_root}/{YYYY-MM-DD}/{YYYY-MM-DD}.01.작업문서.md` |
| 주제별 문서 | `{docs_root}/{YYYY-MM-DD}/{YYYY-MM-DD}.{순번}.{주제}.md` (순번 02~) |

# 처리 절차

## 0단계: 인덱스 체크박스 누락 점검 (전처리)

소스 인덱스의 체크박스 상태와 주제별 문서의 실제 완료 상태를 대조·보정한다.

1. 소스 인덱스의 `> 상세:` 링크가 있는 소주제를 모두 수집
2. 각 주제별 문서를 읽고 체크박스를 파싱
3. **인덱스 미갱신 감지**: 인덱스에 `[ ]`가 있는데 주제별 문서 전체 `[x]` → 인덱스를 `[x]`로 자동 갱신 (경고 출력)
4. **인덱스 누락 감지**: 주제별 문서에 `[ ]` 항목이 있는데 인덱스에 대응 항목 없음 → 인덱스에 `[ ]`로 추가 (경고 출력)
5. **주제별 문서 미갱신 감지 (역방향)**: 인덱스에 `[x]`인 항목이 주제별 문서에서 아직 `[ ]` → 주제별 문서를 `[x]`로 갱신 (경고 출력). 인덱스의 완료 메모(날짜, 비고)가 있으면 함께 반영.

> 주제별 문서에 미완료가 하나라도 있으면 3번은 건드리지 않는다. 링크 없는 소주제는 대상 제외.

## 1단계: 소스 작업문서 탐색

- 인자가 있으면 해당 날짜 작업문서를 찾는다
- 인자가 없으면 오늘 이전 날짜부터 역순으로 작업문서를 탐색한다 (최대 7일)
- 소스 작업문서가 없으면 에러 출력 후 중단

## 2단계: 소스 작업문서에서 이월 대상 추출

### 2-1. "오늘 할 일" 섹션에서 추출

| 상태 | 이월 여부 | 비고 |
|---|---|---|
| `[ ]` | **판단 필요** | 블로킹 여부에 따라 분기 |
| `[→]` | **판단 필요** | 블로킹 여부에 따라 분기 |
| `[-]` | **조건부** | 기본 이월 안 함. 2-3b에서 블로커 해소 감지 시 `[ ]`로 전환 |
| `[x]` / `[완료]` / `[~~]` | 제외 | |

**블로킹 판단**: `블로커:`, `(블로커)`, `회신 대기`, `오픈 진행 중`, `수신 대기` 등 외부 대기 표현이 있으면 "오늘 할 일"에 넣지 않고 "외부 블로커" 테이블에서만 추적.

### 2-2. "작업 목록" 섹션에서 추출

- 각 소주제(### 레벨) 하위의 체크박스를 파싱
- **소주제 전체가 완료·블로킹·취소**이면 통째로 제외
- **실행 가능 항목이 있으면** `[ ]`/`[→]`만 이월 (`[x]`/`[-]`/`[~~]` 제거)
  - 소주제 헤딩·`> 상세:` 링크·설명 텍스트는 함께 이월
  - **"오늘 할 일" 동기화**: P3/P4/잡무가 아닌 이월 소주제는 "오늘 할 일"에도 소주제명 기반 대표 항목을 `[ ]`로 추가 (2-1과 중복이면 스킵, 2-5 네이밍 규칙 적용)

#### 주제별 문서 링크 감지

소주제에 `> 상세:` 줄이 있으면 주제별 문서 링크로 인식한다.

- 이월 대상 소주제의 주제별 문서는 **반드시 오늘 날짜 폴더에 존재**해야 한다. `../` 참조 금지 (이월 제외 소주제는 해당 없음)
- 실행 가능 항목 있음 → 3-3에서 문서 이월 처리
- 인덱스의 `> 상세:` 링크를 오늘 날짜 폴더의 새 파일명으로 갱신

#### 크로스-주제 중복 감지

소주제의 실행 가능 항목이 다른 소주제에서 이미 추적 중인 동일 작업이면 중복 제외. 중복 제외 후 실행 가능 0건이면 이월 제외. 인덱스에 `(→ {원본 주제명}에서 추적)` 표기.

### 2-3. "외부 블로커" 테이블 처리

**2-3a. 미해결 블로커 이월**: 상태가 `완료`/`해소`/`해결`/`회신 완료`가 아닌 행만 이월.

**2-3b. 해소된 블로커 → 재활성화**: 해소된 블로커와 키워드 매칭되는 `[-]`/블로킹 항목을 `[ ]`로 전환하여 "오늘 할 일"에 포함. 보수적 매칭. 출력에 `🔓 블로커 해소 감지` 표시.

### 2-4. 제외 필터

이월하지 않는 패턴: `동료 일임`/`동료 작업`/`(동료)`, `급하지 않음`, `(미머지)`

> **P3/P4, 잡무**: 이월하되 "오늘 할 일"에는 넣지 않고 **작업 목록에만** 이월.
> **블로킹 항목**: "오늘 할 일" 제외. 2-3b에서 해소 감지 시만 포함.

### 2-4a. etc/ 고정 문서 자동 갱신

이월 중 아래 패턴을 감지하면 docs-manager Agent를 **백그라운드**로 호출. 감지 없으면 스킵.

| 감지 패턴 | 동작 |
|---|---|
| 3주+ 이월 항목 (P1/P2 아님) | 사용자에게 미래_계획 이관 여부를 **확인 후** 처리. 임의 이관 금지 |
| 블로커 해소로 미래 계획 선행 조건 충족 | 해당 항목에 `✓ 충족` 표기 |

### 2-5. "오늘 할 일" 항목명 네이밍

CLAUDE.md의 **"오늘 할 일" 항목명 네이밍 규칙**을 적용한다:
- 환경/서비스 식별자가 없으면 소주제명·주제별 문서명에서 추론하여 접두어 보충
- `— {상세설명}` 접미사는 제거
- 호스트 약칭은 핵심 부분만 (예: `app-server-01` → `srv01`)

### 2-6. 이월 태그 부여

- 이월 항목에 `(MM-DD 이월)` 태그 부여
- 이미 태그가 있으면 **원래 이월 시작일을 유지** (날짜 갱신 안 함)

## 3단계: 오늘 작업문서(인덱스) 처리

### 대상(오늘) 문서 체크박스 규칙

> `[→]`는 소스 문서 전용. 대상 문서에 `[→]` 사용 금지.
> "오늘 할 일"에는 실행 가능 항목만 `[ ]`로 넣는다.

### 3-1. 신규 생성

```markdown
---
updated: {YYYY-MM-DD HH:MM}
---
# {YYYY-MM-DD} 작업문서

> **이관 출처:** [{소스 파일명}]({상대경로})의 미완료 항목 이월

## {MM-DD} 오늘 할 일

{이월된 항목들}

## 외부 블로커

| 블로커 | 상태 | 비고 |
|---|---|---|
{미해결 블로커 행들}

## 작업 목록

{이월된 소주제 + 체크박스 + 주제문서 링크}
```

### 3-2. 기존 문서에 추가

- **기존 내용 절대 삭제 금지**
- **중복 방지**: 동일 내용이 이미 있으면 스킵
- "오늘 할 일" 최하단에 이월 항목 추가
- "작업 목록"에 이월 소주제 추가

### 3-3. 주제별 문서 이월

소주제에 `> 상세:` 링크가 있고 이월 대상이면:

**순번 재채번**: 소스 순번을 그대로 쓰지 않는다. 오늘 폴더에 이미 존재하는 파일의 최대 순번 다음부터 연번으로 부여한다 (02, 03, 04, ...).

1. **미진행 판단** (2가지 모두 충족 시):
   - 문서 내 `[x]` 0개, 히스토리 엔트리(`### {HH:MM}`) 0개
   - (`git diff` 변경은 진행 판단에 사용하지 않음 — 오타 수정·날짜 갱신 등은 진행이 아님)
2. **미진행** → `git mv`로 이동+리네임 (재채번된 순번 사용)
3. **진행된 문서** → **doc-filter Agent** 호출하여 필터링 복사:
   - 프롬프트에 소스/대상 경로를 전달
   - Agent 정의: `.claude/agents/doc-filter.md`
4. 인덱스의 `> 상세:` 링크를 새 파일명으로 갱신
5. 이월 제외 주제는 문서 이월 안 함

## 4단계: 소스 작업문서 원본 표기

- `[ ]` → `[→]`로 변경 + 항목 끝에 `(이월)` 추가 (이미 있으면 스킵)
- `git mv`된 문서는 인덱스 체크박스만 `[→]` 처리
- **원본 내용은 절대 수정하지 않음**

## 5단계: updated 타임스탬프 갱신

- 오늘 작업문서의 YAML 헤더 `updated` 값을 현재 시각으로 갱신

# 출력 형식

```
## 이월 완료

소스: `{소스 경로}` → 대상: `{오늘 경로}`

### 이월 요약
- 오늘 할 일: {N}건
- 작업 목록 소주제: {N}건
- 주제별 문서: 필터링 {N}건, git mv {N}건
- 블로커: {N}건
- 제외: {N}건 (해당 시 사유 표시)
- 블로커 해소 재활성화: {N}건 (해당 시)
```

`--dry-run`이면 "이월 미리보기 (dry-run)" 제목, 파일 수정 안 함.

# 제약 조건

1. **기존 내용 절대 삭제 금지** — 추가/삽입만 수행
2. **중복 이월 방지** — 동일 항목이 이미 있으면 스킵
3. **소스 원본 최소 변경** — 체크 상태 변경 + `(이월)` 표기만
4. **CLAUDE.md 제외 규칙 준수** — 동료 담당, P3/P4는 작업 목록에만 이월
5. **이월 시작일 보존** — 기존 `(MM-DD 이월)` 태그 날짜 유지
6. 코드 변경 없음 — 작업문서 파일만 수정
