# 멀티 프로젝트 설계 ("최소 멀티")

> 소스: `ARCHITECTURE.md §멀티 프로젝트 설계`

loregist는 retrofit 비용이 큰 레이어만 멀티로 반영하고 운영 UX는 단일로 유지하는 "최소 멀티" 설계를 따른다. DB 스키마는 멀티(`project` 컬럼 — 나중에 넣으면 전체 재임베딩), 인프라 위치는 중앙화(DB/venv/모델/스크립트 1벌), vault 경로는 멀티(`logvault/{project}/`), 검색 스코프(UX)는 단일(기본 cwd 기준 현재 프로젝트, 크로스는 `--all-projects`)로 배분된다. `project`는 `projects.toml`의 key이며 cwd/대상 경로에서 자동 추론(`infer_project`)되고 `--project`로 override할 수 있다. 새 프로젝트 추가는 `projects.toml`에 블록 추가만으로 완료되어 코드 편집이 불필요하다. catalog(LLM Wiki) 활성화는 블록에 `catalog`·`handbook`·`catalog_readme` 필드를 추가해 opt-in한다.
