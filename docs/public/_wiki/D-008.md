# wiki = handbook + catalog 위계 재정의

> 소스: `plans/done/2026-06-22.wiki_handbook_catalog_위계_재정의.md §접근 방법`

"wiki"라는 단어가 설정 키·스킬·아키텍처 원칙에서 세 가지 다른 의미로 꼬여 쓰이던 것을 해소하기 위해, wiki를 LLM이 유지하는 지식 전체를 가리키는 상위 우산 개념으로만 재정의하고 그 아래를 handbook과 catalog로 분리했다. 사람이 권위를 갖는 수동편집 종합 문서는 `handbook`으로 통일하고(구 설정 키 `wiki`/`wiki_sources` → `handbook`), 자동 증류 색인은 `catalog` 이름·동작 그대로 유지하며 handbook을 입력 소스로 증류하는 관계를 명확히 했다. 스킬 위계도 개념과 동형으로 재구성해, 구 `wiki-update`의 산문 갱신 동작은 `handbook-update`로 분리하고 `wiki-update`는 `handbook-update → catalog-update`를 순차 호출하는 상위 오케스트레이터로 신설했다(증류 방향 = 호출 순서). 개인 repo로 단일 소유라 하위호환이 불필요하므로 deprecated 별칭 없이 즉시 제거하고, "wiki" 식별자는 handbook+catalog 둘 다를 묶는 상위 의미일 때만 잔존을 허용했다.
