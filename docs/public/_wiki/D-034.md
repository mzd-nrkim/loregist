# block_readonly_handbook 대화형 세션 면제 (LOREGIST_AUTO_GUARD 스코핑)

> 소스: `plans/done/2026-06-23.readonly_handbook_훅_대화형_세션_면제.md §배경`

`writable=false` 보호 목적은 stashdex 자동 스킬이 읽기전용 문서를 무인 재작성하지 못하게 막는 것이다. 그런데 `hooks/block_readonly_handbook.py`가 호출 주체와 무관하게 모든 Edit/Write 시도를 차단해, 사용자 대화 세션의 Claude가 사용자 지시로 `plans/` 계획서를 편집하려 할 때도 막히는 문제가 발생했다(실제 발생 → Bash heredoc 우회).

**원칙**: stashdex가 headless로 claude를 자동 실행하는 경우엔 훅이 차단한다. 사용자 대화 세션의 Claude는 훅 영향을 받지 않는다.

**해결**: `LOREGIST_AUTO_GUARD` 환경변수로 자동/대화형을 구분한다. `auto_update.launch_headless()`가 `subprocess.run(["claude","-p", ...])` 호출 시 자식 env에만 `LOREGIST_AUTO_GUARD=1`을 주입하므로, 훅이 이 변수의 존재 여부만 보면 100% 신뢰 가능한 마커가 된다.

**구현**: `main()`의 `_build_blocked_paths()` 호출 이전에 가드 체크를 삽입했다:

```python
if not os.environ.get("LOREGIST_AUTO_GUARD"):
    sys.exit(0)
```

미설정(= 대화형 세션)이면 차단 집합 빌드 비용 없이 즉시 통과. D-012가 도입한 `LOREGIST_AUTO_GUARD` 재귀 가드와 동일 변수를 훅 스코핑에도 재활용해 일관성을 유지한다.
