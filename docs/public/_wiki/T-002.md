# 설계 철학 3원칙 (텍스트 단위·토큰 절약·LLM Wiki)

> 소스: `ARCHITECTURE.md §설계 철학`

loregist 설계는 세 원칙으로 요약된다. ① 텍스트 단위 작업수행 — 모든 업무를 `*.log` 텍스트로 기록해 LLM이 청크 단위로 소비·생산한다(journal·watch·docs-manage 스킬). ② 토큰 절약 — Hot→Cold→vault 계층과 시맨틱 검색으로 top-k 청크만 컨텍스트에 주입한다(loregist search, 검색 계층). ③ LLM Wiki — 텍스트 흐름에서 LLM이 topic·decision을 증류해 지식 베이스를 유지한다(catalog-update 스킬). 핵심 명제는 "모든 업무가 텍스트로 흐르고, 필요한 것만 컨텍스트에 들어오며, 누적된 지식은 LLM이 직접 정제한다"이며, 세 원칙은 순환 파이프라인을 이룬다.
