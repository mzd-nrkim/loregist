# FTS/hybrid 검색 (pg_bigm + RRF k=60)

> 소스: `plans/done/2026-06-11.05.개인repo_FTS_hybrid_검색_확장_계획.md §단계별 접근 (비용 순)`

vector 시맨틱 검색이 "CSMSEL" 같은 정확 키워드를 놓치는 갭을 보완하기 위해, 추가 인프라 비용이 0인 PostgreSQL pg_bigm부터 도입했다. 한국어는 공백 토큰화가 불완전하므로 바이그램(2글자) 인덱싱을 쓰고, tsvector/`ts_rank`는 인덱스·연산이 달라 혼용하지 않는 pg_bigm 단일 메커니즘으로 통일했다. FTS 대상은 원문이 아니라 vector와 단위를 맞추기 위해 청크(`doc_chunks.chunk_text`)로 정했다(원문 단위면 한 원문의 여러 청크에 점수가 중복 부여됨). hybrid는 cosine(0~1)과 bigm 유사도의 스케일 차이로 가중합이 부정확하므로, 순위만 사용하는 RRF(Reciprocal Rank Fusion, `k=60`)로 융합한다. 골든셋 평가 결과 hybrid 80% vs vector 40% / fts 60%로 우월했고, rrf_k는 {10,30,60,100} 모두 동일 적중률이라 기본 60을 유지했다. 형태소 분석이 필요하면 OpenSearch+nori(Step 3)로 넘어가되 별도 계획으로 분리했다.
