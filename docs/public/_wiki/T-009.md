# handbook (writable 산문 종합 문서)

> 소스: `plans/done/2026-06-22.wiki-update_스킬_신규생성_및_catalog-update_분리.md §목표`

handbook은 README.md·ARCHITECTURE.md 같은 writable=true 산문 문서로, `wiki-update` 스킬이 전담 갱신한다. wiki-update는 `_catalog/.last_wiki_update`에 기록된 직전 갱신 커밋 SHA를 base로 삼아 `git log <base>..HEAD` + `git diff <base> HEAD`로 실제 변경을 파악하고, writable=true 파일을 읽어 stale 섹션을 판단한 뒤 섹션 단위 Edit를 제안한다. 사용자 승인으로 1건 이상 반영되면 현재 HEAD SHA를 `.last_wiki_update`에 기록한다. wiki 파일을 쓰는 모든 규칙(update_when 게이트, LOCK 마커 보호, 쓰기 전 승인)은 wiki-update에 단일 귀속되며, catalog-update는 wiki를 일절 쓰지 않는 읽기전용으로 분리된다. 두 스킬의 base 마커는 SHA 1줄 + git diff 방식으로 완전 대칭화되어 있다.
