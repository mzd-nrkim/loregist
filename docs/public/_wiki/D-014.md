# .last_*_update 스탬프 (SHA 기반 diff base)

> 소스: `plans/done/2026-06-23._wiki_flat_구조_도입.md §결정 D0 — stamp 파일 통일 포맷`

Python `loregist catalog`(catalog_gen.py)는 stamp에 ISO 타임스탬프를, LLM `/catalog-update`(SKILL.md)는 커밋 SHA를 써서, 단순 리네이밍 시 같은 파일명에 다른 포맷이 섞여 drift.py의 `datetime.fromisoformat()`이 SHA를 만나 깨지는 충돌이 있었다. 이를 `.last_catalog_update` 하나로 통일하되 `.last_handbook_update`와 대칭(커밋 SHA)으로 맞춰 해소한다. catalog_gen.py는 git repo 안이면 `git rev-parse HEAD` SHA를, 밖이면 ISO 타임스탬프를 조건부로 기록한다. drift.py는 dual-read fallback으로 동작 — 내용이 40-hex SHA면 `git show -s --format=%cI`로 reference 시각을 도출하고, ISO 타임스탬프면 직접 파싱(비-git·구 `.last_update` 하위호환), 둘 다 실패하면 TOPICS/DECISIONS mtime fallback으로 넘어가며 구 `.last_update`도 1회 읽어 마이그레이션한다. 구조화 2필드(`{ts, sha}`) 대안은 복잡도 대비 이득이 적고 handbook과의 대칭성·인지부하 최소화 측면에서 SHA 단일화가 우월해 기각했다.
