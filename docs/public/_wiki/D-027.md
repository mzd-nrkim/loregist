# plan-run 동시실행 충돌 차단 (세션 lock)

> 소스: `plans/done/2026-06-22.planrun_동시실행_충돌차단.md §접근 방법`

여러 창에서 plan-run을 동시에 돌릴 때 워크트리·브랜치가 서로 간섭하는 문제를, 세션 잠금 파일과 세션ID 브랜치 네이밍을 병용해 차단하기로 했다. lock 파일은 `~/.stashdex-planrun.lock`(홈 디렉터리, repo 독립·worktree 공유)에 두어 세션ID·시작시각·대상 계획서·PID를 기록하고, 워크트리는 `wt/<plan-slug>-<session8>` 형태로 만들어 타 세션이 같은 이름을 가로채지 못하게 한다. 머지 직전에는 `git log <branch> ^main`·브랜치 존재 여부·게이트 통과를 재검증해 브랜치가 사라졌거나 이미 머지됐으면 "외부 간섭"으로 보고 중단한다. 다중 창이 의도적일 수 있으므로 hard-block 대신 경고+사용자 확인 정책을 택했고, stale lock(PID 사망 또는 3시간 초과)은 자동 정리한다.
