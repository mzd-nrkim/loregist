# catalog/vault_cleanup opt-in 정책

> 소스: `plans/done/2026-06-19.컨텍스트관리_catalog_vault_후속계획.md §착수 조건`

catalog(주제·결정 정제 인덱스)와 vault_cleanup(오래된 vault 파일 삭제)을 전역이 아니라 프로젝트별 opt-in으로 켜기로 결정했다. catalog 필요성은 전역 검색 품질이 아니라 프로젝트 성격에 따라 다르므로 — 결정·주제가 누적되는 실업무 repo는 정제 인덱스 가치가 높고 고볼륨 원시기록 repo는 검색만으로 충분 — `projects.toml`의 `catalog` 키가 있는 프로젝트에만 적용한다(기본 off). vault_cleanup은 `vault_cleanup` 키로 opt-in하며, 비가역 삭제라 기본 off가 catalog보다 더 중요하다: opt-in(off) + `--apply` 명시 승인이 이중 안전장치를 이룬다. cold/ 파일도 정리 대상에 포함하는데, DB 원문(full_text)이 1차 검색 계층으로 영구 보관되어 파일 삭제가 안전하고, 문서 영속이 필요한 정제 지식은 catalog가 담당하기 때문이다.
