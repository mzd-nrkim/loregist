# projects.toml 프로젝트 설정 스키마

> 소스: `docs/public/USAGE.md §새 프로젝트 추가`

새 프로젝트는 `projects.toml`에 블록을 추가하는 것만으로 온보딩되며 코드 편집이 필요 없다. 경로는 `LOREGIST_WORKSPACE` 기준 상대경로로 작성한다. 일반 프로젝트는 `docs_root`(작업문서)·`vault`(로그)·`cold`(콜드 스토리지)에 더해 `catalog = true`(`_wiki/` 인덱스), `extensions`(임베딩 대상 확장자, 기본 md/log/txt), `vault_cleanup`(오래된 파일 삭제 opt-in), `hot_days`(rotate 기준일 override)를 선언할 수 있다. `plans/done` 로테이션만 필요하면 `vault`+`done`만 선언한다. `handbook`은 감시·참조할 위키 파일 목록으로 문자열 단축형(`path`만, writable=false)과 inline table(`{path, writable, update_when}`)을 혼용할 수 있다. catalog 키 미설정 + handbook 1건 이상 + docs_root 존재 시 `{docs_root}/_wiki`가 암묵 활성화되며, 오프보딩은 블록 삭제 후 `loregist embed`로 처리한다.
