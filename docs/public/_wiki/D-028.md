# wiki/catalog 전수검사 3축 플래그 모드

> 소스: `plans/done/2026-06-22.wiki_catalog_전수검사_생성_강제갱신_모드.md §플래그 설계`

wiki-update·catalog-update 두 스킬의 전수검사·생성·강제갱신 동작을, 혼동을 피하기 위해 서로 독립된 세 축의 플래그로 분리하기로 했다. `--all`은 범위(모든 프로젝트 순회), `--now [이름]`은 게이트(update_when/diff 게이트 무시 + 누락 문서 생성, 값이 있으면 이름 매칭 필터), `--force`는 깊이(기존 문서 통째 재작성)를 담당한다. 누락 생성은 git diff가 없어 게이트로 못 잡으므로 `--now`에 귀속시키고, `--force`는 통째 재작성과 증분 게이트가 모순이므로 자동으로 `--now`를 함의하게 했다. 별개의 `--commit` 옵션은 3축 어느 조합과도 결합 가능하며 쓰기 반영 후 repo별 그룹핑 커밋까지만 수행하고 머지·푸시는 하지 않는다. `--dry-run`은 어떤 쓰기도 없이 대상 목록만 출력한다.
