# catalog drift 신호 + 무인 자동 갱신 (auto_* 플래그)

> 소스: `plans/done/2026-06-22.loregist_실행후_카탈로그갱신_제안.md §접근 방법`

`stashdex embed`는 벡터DB 임베딩만 하고 handbook 갱신·catalog 인덱스 생성은 스킬을 수동 호출해야만 일어나, 새 문서 추가나 writable 변경 시 반영 누락이 발생했다. 이를 두 단계로 메운다. (A) Python이 catalog 마지막 갱신 시각 이후 변경/신규인 handbook을 결정적으로 식별해 `stashdex status`·`embed` 말미·`--json`에 surface하고, drift가 0건이면 어떤 트리거도 발생하지 않아 노이즈를 차단한다. (B) drift가 있을 때 갱신을 수행하되, 방식은 프로젝트별 두 원자 플래그(`auto_handbook_update`·`auto_catalog_update`, 기본 false)로 갈린다 — 둘 다 off면 사후 제안(승인 후 실행), 켜진 조합에 따라 `/handbook-update`·`/catalog-update`·`/wiki-update`를 무인 실행한다. 실행 맥락도 분기해, 세션 안(`CLAUDECODE=1`)이면 현재 에이전트가 PostToolUse hook 리마인더로 수행하고, 독립 실행이면 헤드리스 `claude -p`를 기동한다. 무인 자동이어도 LOCK 보호 영역 미수정·update_when 게이트·사후 리포트 감사는 양보 불가이며, `LOREGIST_AUTO_GUARD`로 재귀·이중 실행을 차단한다.
