# 로그를 repo 밖 vault로 분리

> 소스: `plans/done/2026-06-11.04.개인repo_문서_로그_컨텍스트_관리_수정계획.md §확정 사항`

`.gitignore`로는 Claude의 파일시스템 직접 읽기를 막을 수 없고 CLAUDE.md 규칙은 soft boundary라 구조적 보장이 안 된다는 인식 아래, cold 파일을 repo 밖으로 빼는 방식(로그 배치 규칙 B)을 채택했다. `.log` 등 누적 기록을 `~/workspace/logvault/{project}/{YYYY-MM-DD}/`로 이동시켜 repo의 Grep/Glob 검색 표면에서 구조적으로 제거하고, 그 vault를 vector DB 임베딩 원본 소스로 활용한다. vault 경로는 CLAUDE.md에 기재하지 않아 Claude가 자발적으로 탐색하지 않게 한다. 인프라(DB·venv·모델·스크립트)는 repo 밖 중앙 1벌로 두고 각 repo는 호출만 갖되, DB 스키마에는 `project` 컬럼을 넣어 멀티 프로젝트 분리를 retrofit 비용이 큰 레이어에서만 미리 반영했다. 원문은 `doc_originals.full_text`에 보존되어 vault 삭제 후에도 복원 가능하다.
