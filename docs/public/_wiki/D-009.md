# catalog-update / wiki-update 책임 분리

> 소스: `plans/done/2026-06-22.wiki-update_스킬_신규생성_및_catalog-update_분리.md §접근 방법`

`catalog-update` 스킬이 wiki 파일 직접 갱신과 catalog 인덱스 생성이라는 두 책임을 겸해 역할이 모호했다. 이를 해소하기 위해 wiki 파일을 *쓰는* 모든 규칙(update_when 게이트·LOCK 보호·쓰기 전 승인)을 신규 `wiki-update` 스킬에 단일 귀속시키고, `catalog-update`는 wiki를 일절 쓰지 않는 읽기전용 인덱스 생성 전용으로 단순화한다. wiki-update는 `.last_wiki_update`(커밋 SHA 1줄)로 base를 정해 `git diff <base> HEAD`로 변경을 파악하고, writable=true 파일의 stale 섹션을 섹션 단위로 Edit 제안한다. 두 스킬의 base 마커는 `.last_wiki_update`·`.last_catalog_update`로 네이밍·SHA 값·diff 메커니즘이 완전 대칭이며, 각자 자기 마커만 읽고 쓴다. 별도 계획이던 update_when 게이트(P3)·writable=false 차단(P5)의 대상 스킬도 모두 catalog-update에서 wiki-update로 이동했다. 순수 SKILL.md LLM 명세라 코드를 동반하지 않아, 검증은 명세 일치 확인과 `claude -p` 라우팅 테스트로 대체했다.
