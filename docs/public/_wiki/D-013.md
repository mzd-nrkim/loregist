# catalog 인덱스 파일 embed 포함

> 소스: `plans/done/2026-06-22.catalog_인덱스파일_embed_제외_결정_정정.md §목표`

직전 계획에서 `TOPICS.md`·`DECISIONS.md`를 embed에서 제외하는 필터를 넣었으나, 사후 검증 결과 이 결정이 틀렸음을 확인하고 정정한다. `_catalog/`에는 개별 T-xxx·D-xxx 항목 파일이 0건이고 두 인덱스가 catalog의 유일한 콘텐츠라, 제외하면 catalog 임베딩이 0건이 되어 기능 자체가 무의미해진다. 또한 두 인덱스는 `id|요약|관련문서`, `date|결정|근거|related` 형태로 결정·근거·관련문서가 한 행에 묶여 검색 진입점으로서 일반 md보다 가치가 높고, search.py가 source_kind로 WHERE 필터를 하지 않아 포함 시 일반 청크와 동일하게 검색된다. 따라서 `discover_embed_files()`의 제외 필터를 제거해 `_catalog/`의 모든 `.md`를 `source_kind="catalog"`로 포함시키고, 빈 AUTO 영역만 있어도 마커 밖 표가 정상 청킹되며 순수 템플릿은 기존 빈 파일 가드로 안전 처리됨을 확인했다. 테스트도 "인덱스 제외"에서 "인덱스 포함 + kind=='catalog'"로 뒤집었다.
