# 계획서 테스트 섹션 표준 구조

> 소스: `plans/done/2026-06-15.계획서_테스트_누락_재발방지_메타계획.md §2. 해결책`

테스트 누락이 반복되는 문제를 막기 위해, 모든 구현 계획서가 따라야 하는 표준 구조에 `## 테스트 계획` 섹션을 삭제 불가 필수 섹션으로 신설하기로 했다. 이 섹션은 수행 단계보다 앞에 위치하며 단위 테스트(RIGHT-BICEP), 실DB 통합 테스트(real_db fixture), E2E/CLI 테스트, 회귀 검사의 네 항목을 명시하도록 표준화한다. 완료 조건에도 "make test-int 통과(기존 regression 0건 + 신규 TC 전부 green)"를 삭제 불가 항목으로 박아, 테스트 작성·통과가 계획서 구조 차원에서 강제되도록 했다. 더불어 작성 전 자가 체크리스트(테스트 계획 섹션 존재·함수별 RIGHT-BICEP·통합 시나리오·CLI E2E·완료 조건)를 두어 누락을 사전에 차단한다. 구조적 강제로 soft한 규칙 의존을 대체한 것이 핵심 트레이드오프다.
