# writable=false 코드레벨 차단 (심층 방어)

> 소스: `plans/done/2026-06-22.writable_false_코드레벨_차단.md §접근 방법`

`writable=false` wiki 파일의 "수정 금지"가 오직 SKILL.md 프롬프트 규칙(제약 #8)에만 의존해, LLM이 규칙을 어기면 막을 기계적 방어가 없었다. writable=false 조건 집합은 유한(파일 경로 목록)하므로 코드/하네스 레벨에서 1차 차단이 가능하다는 점에 착안해, "프롬프트 1차 의존"에서 "코드 1차 + 프롬프트 2차"의 심층 방어(defense in depth)로 전환한다. 메커니즘은 (가) PreToolUse hook으로 Edit/Write의 file_path를 차단 집합과 대조하는 하드 차단과 (나) 스킬 helper가 writable=true 경로만 LLM에 노출하는 노출 차단을 비교한 뒤, 두 방식을 병행하는 (다)를 채택했다. 과차단을 막기 위해 차단 스코프는 쓰기 주체인 wiki-update 워크플로 컨텍스트로 한정하고, writable 판정 소스는 `loregist project list --json`으로 일원화하며 경로 정규화(realpath)로 우회를 차단한다. 파일 내부 영역인 LOCK 보호는 file_path 단위로 부족해 후속 분리로 미뤘고, 차단·통과·과차단 시나리오는 pytest로 회귀 검증한다.
