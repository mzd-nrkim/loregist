# 기술 스택 (pgvector·e5-small·검색모드·Python)

> 소스: `ARCHITECTURE.md §아키텍처`

stashdex의 기술 스택은 네 축으로 구성된다. DB는 PostgreSQL 16 + pgvector로 port 5433, db/user는 `loregist`/`loregist`를 쓴다. 임베딩 모델은 `dragonkue/multilingual-e5-small-ko-v2`(384 dims, 약 450MB, 한국어 특화 fine-tune)이다. 검색 모드는 `hybrid`(RRF 융합, 기본)·`vector`·`fts` 세 가지이며, hybrid가 골든셋 80%로 단독 40~60%보다 종합 우월하다. 런타임은 Python 3.11(`.python-version`)에 venv `.venv/`를 사용한다. 컴포넌트는 `config.py`(레지스트리 로드·DB 접속), `chunking.py`(split), `embed.py`, `search.py`, `rotate.py`, `tui.py` 등으로 나뉘고 스키마는 `infra/init.sql`의 `doc_originals`/`doc_chunks` + ivfflat·project 인덱스로 정의된다.
