# source_kind 분류 (md/log/catalog/wiki)

> 소스: `plans/done/2026-06-22.wiki_sources_embed_검색_파이프라인_통합.md §목표`

`source_kind`는 DB에서 문서를 출처별로 구분하는 분류 값으로 md/log/catalog/wiki 네 종류로 정리됐다. 직전 작업에서 `_catalog`(구조화 출력 축)는 embed 통합이 완료됐으나 `wiki_sources`(자유형 입출력 축)는 embed·검색 파이프라인에 미연결이라 일관성이 깨져 있었다. 이를 보완해 `wiki_sources`에 선언된 경로(절대경로/glob 확장 포함)가 embed 대상에 포함되고 별도 `source_kind="wiki"`가 부여되어 _catalog/md/log와 구분된다. watch 자동 감시 범위에도 `wiki_sources` 경로가 포함된다. `search.py`는 `source_kind`로 WHERE 필터하지 않으므로 embed만 되면 검색은 자동으로 동등 취급되며, 결과적으로 wiki_sources와 _catalog가 embed·검색에서 동등하게 취급되어 기능 일관성이 완성된다.
