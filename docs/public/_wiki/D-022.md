# projects.toml extensions 설정화

> 소스: `plans/done/2026-06-19.projects_toml_확장자_설정_추가.md §목표`

텍스트 파일 인식 확장자(`.md`·`.log`)가 `watch.py`와 `embed.py`에 분산 하드코딩되고 경로별로 다르게 적용되던 것을, `projects.toml`의 `extensions` 필드로 프로젝트별 선언화하기로 결정했다. `ProjectConfig`에 `extensions: list[str]` 필드를 추가하고 기본값 `["md", "log", "txt"]`을 두어, 미지정 시 기존 동작이 유지된다. `watch.py`·`embed.py`·`vault_cleanup.py`가 하드코딩 대신 config에서 확장자 목록을 읽도록 교체해, 경로별(vault/docs_root/cold/done) 분기 로직을 config 기반 단일 로직으로 통일했다. 이로써 watch와 embed가 동일한 `PROJECTS[project]`를 참조해 감시 범위와 임베딩 범위 불일치가 해소되고, 사용자는 코드 수정 없이 `.txt` 등 확장자를 추가할 수 있다.
