# update_when 갱신 판단 게이트

> 소스: `plans/done/2026-06-22.update_when_갱신판단_게이트화.md §접근 방법`

`update_when`은 본래 갱신을 제안할 때 출력하는 근거 문구에 불과해, 값이 있어도 실제 갱신 여부·시점에는 영향을 주지 않았다. 이를 writable 파일을 *언제* 갱신할지 판단하는 게이트로 승격한다. update_when 값이 있으면 LLM은 갱신 제안 전에 "현재 변경 맥락이 이 조건에 부합하는가"를 먼저 판정해, 부합 시에만 갱신 제안과 근거 문구를 출력하고 불부합 시 갱신 대상에서 제외(스캔만)한다. update_when이 없으면 게이트 없이 일반 staleness 판단을 유지해 하위호환을 보장한다. 게이트를 통과해도 LOCK 보호 영역과 사용자 승인 규칙은 그대로 적용된다. 다만 이 계획은 후속 결정에서 catalog-update가 wiki를 쓰지 않는 읽기전용으로 바뀜에 따라 게이트의 귀속처가 wiki-update로 이동했고, 본 계획 자체는 P4(D-009)에 흡수되어 별도 실행하지 않는다.
