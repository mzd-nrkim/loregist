# archive 용어 → cold/done 분리

> 소스: `plans/done/2026-06-18.archive_용어정리_cold_done_계획.md §목표 명칭 체계`

`archive`라는 단어가 toml 키·코드·폴더·rotate 목적지·함수명에 중복 등장하며 두 성격(이미 vault에 안착한 cold storage vs repo 내 완료문서 rotate 대기)을 겸해 의미가 충돌했고, 그 결과 rotate가 cold storage를 다시 rotate 대상으로 오인하는 버그까지 발생했다. 이를 두 키로 분리한다 — `cold`(`logvault/{p}/cold`)는 cold storage 종착지로 `*.md`를 embed만 하고 rotate 비대상이며, `done`(`plans/done`)은 완료문서로 embed + rotate 대상(파일명 날짜 기준)이고 목적지는 `vault/cold/`다. 그 결과 라이프사이클이 용어 자체로 흐름으로 읽힌다(docs_root/{날짜} → vault/{날짜}, plans/done → vault/cold). 정리 적기인 이유는 logvault가 git 밖이라 rename에 git 영향이 없고, rotate 목적지가 아직 미생성이라 비용이 최소이며, DB는 `UPDATE replace(source_path,...)`로 재임베딩 없이 경로만 교체 가능하기 때문이다. 폴더 rename → DB UPDATE → toml → 코드 → 테스트 순서를 절대 준수한다.
