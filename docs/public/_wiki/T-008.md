# 문서 라이프사이클 키 체계 (docs_root·vault·cold·done)

> 소스: `docs/public/USAGE.md §라이프사이클`

문서는 rotate와 vault_cleanup 두 단계로 라이프사이클을 관리한다. `stashdex rotate`는 hot 파일 중 일정 기간이 지난 것을 vault(cold)로 이동하며, 기준일은 기본 7일(`ROTATE_TO_VAULT_DAYS`)이고 `projects.toml`에 `hot_days`를 선언하면 해당 프로젝트에만 적용된다. `vault_cleanup`은 rotate로 이동된 파일을 일정 기간 후 삭제하는 opt-in 전용 기능으로, `projects.toml`에 `vault_cleanup` 키(`true`=기본 90일, 또는 일수 직접 지정)를 추가한 프로젝트만 동작하고 `--apply` 명시가 있어야 비가역 삭제가 실행된다. DB의 `doc_originals.full_text`(원문)는 삭제 후에도 보존되므로 원문 복원이 가능하다. 키 의미상 `cold`는 이미 cold storage 종착지라 rotate 비대상이고, `done`은 `plans/done` 파일을 날짜 기준으로 vault로 이동하는 대상이다.
