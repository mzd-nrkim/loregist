# 데이터 플로우 파이프라인 (doc_originals/doc_chunks)

> 소스: `ARCHITECTURE.md §아키텍처`

업무 활동이 로그로 들어와 데이터로 변환되고 소비·증류·회수되기까지의 end-to-end 흐름이다. 기록 단계(journal/watch/digest 스크립트)에서 `*.log`가 생성되고, 임베딩 단계에서 `split_log`/`split_md`로 청크 분할 후 전문은 `doc_originals`에 upsert, 384차원 청크는 `doc_chunks`(pgvector)에 insert된다. 검색 단계는 쿼리를 임베딩해 cosine top-k로 청크를 반환하며 hybrid는 RRF로 융합한다. 이후 스킬이 top-k 청크를 컨텍스트로 소비하고, catalog-update가 로그에서 topic·decision을 `_wiki/`로 증류하며, rotate가 Hot(repo)을 vault(Cold)로 회수한다. 핵심 비대칭은 "다량의 원시 로그는 검색 랭킹으로, 소수의 정제 지식은 LLM 위키로" — 같은 텍스트 흐름의 두 끝이다.
