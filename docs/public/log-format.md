# 로그 형식 가이드

stashdex가 문서를 시맨틱 청크로 분리하는 규칙(`split_log`)을 기준으로 한 최소 권장 형식입니다.  
이 형식을 따르면 검색 정확도가 높아집니다.

## 청킹 규칙 요약

| 항목 | 값 |
|------|-----|
| 분리 기준 | 빈 줄(`\n\n`) |
| 최소 청크 크기 | 100자 |
| 최대 청크 크기 | 1500자 |
| `##` 섹션 구분 | 의미 단위로 나뉘어 검색 정확도 향상 |

빈 줄로 구분된 단락이 100자 미만이면 다음 단락과 병합됩니다.  
1500자를 초과하는 단락은 여러 청크로 분리됩니다.

## 권장 형식

```markdown
## 2026-06-19

- API 스펙 검토 완료: v2 엔드포인트 유지 결정
- 다음 할 일: 백엔드 팀과 v2 싱크 일정 잡기
- 블로커: 인증 모듈 미확정

## 미팅 메모 — 09:30 기획 싱크

참석: A, B, C
결론: 모바일 우선, 데스크탑은 Q3
```

- `##` 섹션 구분이 있으면 의미 단위로 나뉘어 검색 정확도가 올라갑니다.
- 단락은 빈 줄(`\n\n`)로 구분하세요.
- 각 단락은 100자 이상이 되도록 작성하면 독립 청크로 인덱싱됩니다.

## `stashdex journal` 사용 시

`stashdex journal "메모 내용"` 명령으로 기록하면 `[HH:MM]` 접두사가 자동으로 붙습니다:

```
[09:30] API 스펙 검토 완료, v2 엔드포인트 유지 결정
[11:15] 백엔드 팀 싱크 완료 - 다음 주 화요일 배포 확정
[14:00] 블로커 해소: 인증 모듈 담당자 배정됨
```

`[HH:MM]` 접두사는 분리 기준이 명확해져 단락별로 별도 청크로 인덱싱되는 데 도움을 줍니다.

## .log 파일 직접 작성 시

```
[09:30] 오늘 기획 미팅에서 결정된 사항: 모바일 우선 개발, 데스크탑은 Q3로 연기.
참석자: 기획팀 A, B / 개발팀 C, D. 후속 작업: 와이어프레임 작성(기획팀, 6/24까지).

[11:00] API 스펙 v2 검토 완료. 기존 /api/v1 엔드포인트는 6개월 유지 후 deprecated.
변경사항은 변경로그 문서에 기록 예정.
```

- 빈 줄로 단락을 구분하면 각 단락이 독립 청크로 검색됩니다.
- 타임스탬프 `[HH:MM]` 형식을 유지하면 시간 순서 파악이 쉬워집니다.
